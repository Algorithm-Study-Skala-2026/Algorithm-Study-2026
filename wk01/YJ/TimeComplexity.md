## 1. 시간복잡도

> 시간 복잡도 : 주어진 문제를 해결하기 위한 연산 횟수
> 

| 표기법 | 의미 | 기준 | 설명 |
| --- | --- | --- | --- |
| **Big-O (O)** | 최악의 경우 | 상한 (Upper Bound) | 알고리즘이 **이 이상은 걸리지 않는다**는 의미
시간 복잡도를 말할 때 가장 많이 사용 |
| **Big-Ω (Ω)** | 최선의 경우 | 하한 (Lower Bound) | 알고리즘이 **적어도 이만큼은 걸린다**는 의미 |
| **Big-Θ (Θ)** | 정확한 차수 | 상한 = 하한 | 최악과 최선의 차수가 같을 때 사용
알고리즘의 실제 성장률을 정확히 표현 |

```python
import random
findNumber = random.randrange(1,101)

for i in range(1,101):
	if i == findNumber:
		print(i)
		break
```

<aside>
➡️

**시간 복잡도**

- Ω(1) → best case
- Θ(n/2) → average case
- O(n) → worst case

* n : 데이터 개수

</aside>

**1. Big-O (빅오, O 표기법)**

- **의미**: 최악의 경우(upper bound, 상한)의 연산 횟수를 나타낸 표기법
- 알고리즘이 **최대 이 정도 시간은 걸린다**는 의미
- 가장 많이 사용하는 표기법
- 입력 크기 `n`이 커질 때 **성장 속도의 상한**을 표현
- 보통 "시간 복잡도"라고 하면 대부분 Big-O를 의미

---

**2. Big-Ω (빅오메가, Ω 표기법)**

- **의미**: 최선의 경우(lower bound, 하한)의 연산 횟수를 나타낸 표기법
- 알고리즘이 **최소 이 정도 시간은 걸린다**는 의미
- 입력이 가장 좋은 상황일 때의 수행 시간

---

**3. Big-Θ (빅세타, Θ 표기법)**

- **의미**: 보통일 때(상한 + 하한이 같은 경우, average case)의 연산 횟수를 나타낸 표기법
- 최악과 최선이 같은 차수일 때 사용
- 알고리즘의 **정확한 시간 복잡도**

---

## 2. 코딩 테스트에서 사용하는 시간 복잡도 유형

- O(n) 표기법을 기준으로 수행 시간을 계산하는 것이 좋음
- 여러 개의 다양한 테스트 케이스를 수행해 모든 케이스를 통과해야함
- ⇒ 최악의 경우를 염두해야 한다

![image.png](attachment:6dede7e3-ad10-4c31-b157-c5d18b6410d5:image.png)

→ **데이터의 크기(N)**의 증가에 따라 성능이 달라진다

### 2-1. 연산 횟수 계산하여 알고리즘 선택

- 연산 횟수 = 알고리즘 시간 복잡도 n값에 데이터의 최대 크기를 대입하여 도출
- 제한 시간 : 2초 라면,
    
    → 연산 횟수 : 4000만 번 이하(대략 1초에 2000만 번 연산하는 것을 기준으로 생각)
    

<aside>
➡️

알고리즘 적합성 평가

- $n$ → (1 ≤ N ≤ 1,000)
- 버블 정렬 (O($n^2$)) → $(1,000,000)^2$ > 4천만
- 병합 정렬(O($nlog_2n$) → $1,000,000log_2(1,000,000)$ < 4천만
- ⇒ 병합 정렬 = 적합 알고리즘
</aside>
