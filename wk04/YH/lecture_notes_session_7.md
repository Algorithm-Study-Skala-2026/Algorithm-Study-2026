<섹션 7> 그래프

1. 그래프 기본
- 그래프: 노드와 에지로 구성된 집합.
- 노드: 데이터를 표현하는 단위 / 에지: 노드를 연결


2. 그래프의 표현(3가지)

2-1. 에지 리스트(edge list): 에지를 중심으로 그래프 표현. 배열에 출발 노드, 도착 노드를 저장하여 에지를 표현. 또는 출발 노드, 도착 노드, 가중치를 저장하여 가중치가 있는 에지를 표현.
(1) 가중치 없는 그래프 표현
- 출발 노드와 도착 노드만 표현하므로 배열의 행은 2개면 충분함.
- 노드는 여러 자료형을 사용할 수 있음.
- 방향이 있는 그래프는 순서에 맞게 노드를 배열하여 저장하는 방식으로 표현. 방향이 없는 그래프의 경우 순서 상관 없음(같은 표현)
(2) 가중치 있는 그래프 표현
- 그래프의 행을 3개로 늘려 3번째 행에 가중치 저장.
- ex) 1에서 2로 향하는 가중치가 8인 에지: [1, 2, 8]
- 특정 노드와 관련되어 있는 에지를 탐색하기는 쉽지 않음.
- 에지 리스트는 벨만 포드나 크루스칼 알고리즘에 사용하며, 노드 중심 알고리즘에는 잘 사용하지 않음.

2-2. 인접 행렬
- 2차원 배열의 자료구조를 이용하여 그래프 표현, 노드 중심으로 그래프 표현.
- 두 노드를 연결하는 에지의 여부와 가중치 값은 배열에 직접 접근하면 바로 확인 가능
- 노드와 관련되어 있는 에지를 탐색하려면 N번 접근해야하므로 노드 개수에 비해 에지가 적을 때는 공간 효율성이 떨어짐.
- 노드 개수가 많은 경우 아예 2차원 배열 선언 자체를 할 수 없는 결함 있음.

2-3. 인접리스트
- Array List로 그래프를 표현. 노드 개수만큼 Array List를 선언.
- 가중치가 있는 경우 자료형을 클래스로 사용
- 다른 방법에 비해 그래프 구현이 복잡한 편
- 노드와 연결되어 있는 에지를 탐색하는 시간이 매우 뛰어남.
- 노드개수가 커도 공간 효율이 좋아 메모리 초과 에러 발생 X


3. 유니온 파인드(Union-Find)
- 일반적으로 여러 노드가 있을 때 특정 2개의 노드를 연결해 1개의 집합으로 묶는 union 연산과, 두 노드가 같은 집합에 속해있는지를 확인하는 find 연산으로 구성되어 있는 알고리즘.
- 표현: 1차원 배열을 이용. 처음에는 노드가 연결되어 있지 않으므로 각 노드가 대표 노드. → 배열은 자신의 인덱스 값으로 초기화.
- 2개의 노드를 선택해 각각의 대표노드를 찾아 연결하는 union 연산 수행
- 항상 대표 노드끼리 연결!!
- find 연산은 자신이 속한 집합의 대표 노드를 찾는 연산.
- 단순히 대표노드 찾는것 뿐만 아니라 그래프를 정돈하고 시간 복잡도를 향상 시킴.
- 대표 노드에 도달하면 재귀함수를 빠져나오면서 거치는 모든 노드 값을 루트 노드값으로 변경.


4. 위상 정렬(Topological Sort)
- 사이클이 없는 방향 그래프에서 노드 순서를 찾는 알고리즘
- 기능: 노드 간의 순서를 결정 / 특징: 사이클 이 없어야 함.
- 시간 복잡도: O(V+E), V=노드 수, E: 에지 수
- 진입 차수: 자기 자신을 가리키는 에지의 수


5. 다익스트라(Dijkstra)
- 그래프에서 최단 거리를 구하는 알고리즘
- 기능: 출발 노드와 모든 노드 간의 최단 거리를 탐색.
- 특징: 에지가 모두 양수여야 함
- 시간 복잡도: O(ElogV)
- 인접 리스트를 선택하여 구현하는 것이 좋음.
- 최단 거리 배열을 만들고 출발 노드는 0, 나머지는 무한으로 초기화.
- 값이 가장 작은 노드를 고름(값이 0인 출발 노드에서 시작)
- 선택된 노드에 연결도니 에지의 값을 바탕으로 다른 노드의 값을 업데이트. 지정해놓은 연결 리스트를 이용해 현재 선택된 노드의 에지들을 탐색하고 업데이트. 연결 노드의 최단거리는 두 값중 더 작은 값으로 업데이트.

6. 벨만-포드(Bellman-Ford)
- 최단 거리를 구하는 알고리즘.
- 기능: 특정 출발 노드에서 다른 모든 노드까지의 최단 경로를 탐색
- 특징: 음수 가중치 에지가 있어도 수행 가능. 전체 그래프에서 음수 사이클의 존재 여부 판단 가능.
- 시간 복잡도: O(VE)
- 에지를 중심으로 동작하므로 그래프를 에지 리스트로 구현.
- 최단 경로 리스트를 출발 노드는 0, 나머지는 무한대로 초기화.
- 업데이트 반복횟수 = 노드 개수 - 1
- D[s] != 무한대, D[e] > D[s] + w 일 때 D[e] = D[s] + w로 리스트 값 없데이트
- 음수 사이클 유무 확인 위해 모든 에지를 한 번씩 다시 사용해 업데이트되는 노드가 발생하는지 확인.

7. 플로이드-워셜(Floyd-Warshall)
- 그래프에서 최단 거리를 구하는 알고리즘
- 기능: 모든 노드 간에 최단 경로 탐색
- 특징: 음수 가중치 에지가 있어도 수행 가능. 동적 계획법의 원리를 이용해 알고리즘에 접근.
- 시간 복잡도: O(V^3)
- A 노드에서 B노드까지 최단 경로를 구했다고 가정했을 떄 최단 경로 위에 K노드가 존재한다면 rmrjtdfm 이루는 부분 경로 역시 최단 경로임.
- 점화식: D[S][E] = Math.min(D[S][E], D[S][K] + D[K][E])
- for 경유지 K에 관해 (1~N)
	for 출발 노드 S에 관해 (1~N)
		for 도착 노드 N에 관해(1~N)
			D[S][E] = Math.min(D[S][E], D[S][K] + D[K][E])
- 이 3중 for문의 형태 반복하며 리스트 값 업데이트

8. 최소 신장 트리(MST)
- 그래프에서 모든 노드를 연결할 때 사용된 에지들의 가중치의 합을 최소로 하는 트리.
- 사이클이 포함되면 가중치의 합이 최소가 될 수 없으므로 사이클을 포함하지 않음.
- N개의 노드가 있으면 최소 신장 트리를 구성하는 에지의 개수는 항상 N - 1개.
- 데이터를 노드가 아닌 에지 중심으로 저장하므로 에지 리스트의 형태로 저장.(노드 변수 2개와 가중치 변수로 구성)
- 사이클 처리를 위한 유니온 파인드 리스트도 함께 초기화.
- 에지 리스트에 담긴 그래프 데이터를 가중치 기준으로 오름차순 정렬.
- 가중치 낮은 에지부터 순서대로 선택해 연결을 시도. 이때 바로 연결하지 않고 이 에지를 연결했을 때 그래프에 사이클 형성 유무를 find 연산을 이용해 확인한 후 사이클이 형성되지 않을 때만 union 연산을 통해 연결.
